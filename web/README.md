# Jarvis Eel Web Interface v2.0

Улучшенный веб-интерфейс для управления приложением Jarvis Eel. Этот сервер позволяет управлять вашим Jarvis Eel приложением через современный веб-интерфейс с расширенными возможностями мониторинга и управления.

## 🚀 Новые возможности (v2.0)

- Улучшенный веб-интерфейс с русским языком
- Расширенное управление процессом Jarvis Eel
- Реальный мониторинг логов и истории операций
- Системная информация и статистика
- Автоматическое обновление состояния
- Улучшенная безопасность и обработка ошибок

## ⚙️ Установка

1. Убедитесь, что у вас установлен [Node.js](https://nodejs.org/) (версия 14 или выше)
2. Убедитесь, что у вас установлен Python и Jarvis Eel работает корректно
3. Установите зависимости: `npm install`

## ▶️ Использование

### Запуск сервера
```bash
npm start
```
или
```bash
node server.js
```

После запуска откройте в браузере: `http://localhost:3000`

### Основные функции интерфейса

- **Запустить Jarvis Eel**: Запускает процесс jarvis_eel.py
- **Остановить Jarvis Eel**: Останавливает работающий процесс
- **Отправить команду**: Отправляет команду в Jarvis Eel
- **Получить системную информацию**: Отображает информацию о системе
- **Обновить статус**: Обновляет текущее состояние процесса
- **Очистить логи**: Очищает только последние логи
- **Очистить историю**: Очищает всю историю операций
- **Авто-скролл**: Включить/выключить автоматическую прокрутку логов

## 📊 Мониторинг

Интерфейс предоставляет следующую информацию:
- Состояние процесса (запущен/остановлен)
- PID запущенного процесса
- Количество записей в логах
- Количество записей в истории
- Системная информация (платформа, память, загрузка CPU)

## 🔧 API эндпоинты

- `GET /` - Главная страница интерфейса
- `POST /api/start` - Запуск Jarvis Eel
- `POST /api/stop` - Остановка Jarvis Eel
- `GET /api/status` - Получение статуса Jarvis Eel
- `POST /api/command` - Отправка команды Jarvis Eel
- `GET /api/logs` - Получение последних логов (с параметрами limit/offset)
- `GET /api/history` - Получение истории операций (с параметрами limit/offset)
- `POST /api/logs/clear` - Очистка последних логов
- `POST /api/history/clear` - Очистка всей истории
- `GET /api/health` - Проверка работоспособности сервера
- `GET /api/system` - Получение информации о системе

## 🛠️ Требования

- Node.js (v14 или выше)
- Python (должен быть доступен в PATH)
- Jarvis Eel (jarvis_eel.py) должен находиться в родительской директории

## 🔍 Устранение неполадок

Если сервер не может запустить Jarvis Eel:
1. Проверьте, что Python установлен и доступен в PATH
2. Проверьте, что файл jarvis_eel.py находится в родительской директории (../jarvis_eel.py)
3. Проверьте права доступа к файлам
4. Проверьте, что порт 3000 не занят другим приложением

## 🔄 Обновление с предыдущей версии

Если вы обновляетесь с версии 1.x:
1. Замените файлы server.js, index.html, package.json и README.md
2. Выполните `npm install` для установки новых зависимостей
3. Запустите сервер командой `npm start`
